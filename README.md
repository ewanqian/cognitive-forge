## 🚀 项目简介

**认知锻造炉**是一套基于Trae Solo Agent + Obsidian的智能知识管理与认知增强系统，它将知识库管理、红蓝对抗训练、认知增强等功能融为一体，帮助你构建系统化的知识库，淬炼强大的思维能力，应对信息时代的各种挑战。

### 📝 项目定位

- **中文名称**：认知锻造炉
- **英文名称**：Cognitive Forge
- **项目口号**：锻造你的知识库，淬炼你的思维力
- **核心价值**：释放大脑资源，提升认知能力，加速自我成长

### 🧠 智能知识库管理

- 本地安全存储，数据完全可控

### ⚔️ 红蓝对抗训练
- 模拟假想敌，多维度挑战你的思维
- 发现潜在漏洞，优化决策质量
- 危机应对训练，提升应变能力
- 决策优化支持，减少思维盲区

### 🚀 认知增强
- 外部记忆系统，释放大脑认知资源
- 控制论反馈机制，持续优化思想

- 创意激发和联想能力提升

### 🔒 本地安全

## 🔌 MCP功能介绍

### 什么是MCP？
MCP（Model Context Protocol）是Trae AI的"外挂接口"，允许AI智能体连接外部工具和数据源，扩展其能力边界。通过MCP，Trae可以调用搜索、数据库、自动化脚本等外部资源，实现更复杂的任务处理。

### MCP在认知锻造炉中的应用

1. **知识库增强**
   - 连接外部数据库，扩展知识来源
   - 调用搜索API，实时获取最新信息
   - 集成文档处理工具，自动化整理知识

2. **红蓝对抗升级**
   - 接入多模型API，实现更强的对抗能力
   - 调用数据分析工具，更全面评估决策风险
   - 集成模拟环境，实现更真实的危机演练

3. **认知增强扩展**
   - 连接学习平台，实现个性化学习推荐
   - 调用创意生成工具，激发更多灵感
   - 集成心理健康工具，支持思维健康管理

4. **工作流自动化**
   - 连接办公软件，实现文档自动生成
   - 调用日程管理工具，优化时间安排
   - 集成项目管理工具，提升工作效率

### 推荐MCP插件

| 插件类型 | 插件名称 | 功能描述 | 应用场景 |
|---------|---------|---------|---------|
| 多模型切换 | 官方认证多模型切换增强 | 支持智谱AI、MoonShot等国内大模型 | 需要不同模型能力时 |
| VSCode集成 | Trae VSCode插件 | 作为MCP客户端，用于代码优化与调试 | 编程和技术文档编写 |
| 代码优化 | ESLint插件 | 用于代码优化与调试 | 技术知识库管理 |
| 设计工具集成 | Sketch/Figma MCP | 支持设计稿生成到自动化部署 | 设计类知识库管理 |
| 地图服务 | 高德地图MCP | 支持行程规划等应用场景 | 旅行、物流等领域知识库 |
| 流量分析 | WireMCP服务器 | 通过实时网络流量分析增强LLMs能力 | 网络安全、数据分析领域 |
| 平台协同 | 火山引擎MCP | 与火山引擎等平台协同工作 | 企业级应用场景 |

### MCP使用建议

1. **根据需求选择插件**：根据你的知识库主题和使用场景，选择适合的MCP插件
2. **合理配置插件**：按照插件文档正确配置，确保最佳使用效果
3. **注意资源消耗**：部分MCP插件可能消耗较多资源，建议根据需要开启
4. **定期更新插件**：保持插件更新，获取最新功能和安全修复
5. **结合本地功能使用**：MCP功能是本地功能的补充，建议结合使用以获得最佳体验

## 🚀 快速入门指南

### 第一阶段：环境搭建（1-2天）
1. 下载并安装Trae Solo Agent
2. 下载并安装Obsidian
3. 配置Trae访问你的工作文件夹
4. 安装Obsidian必要插件（如Graph View、Quick Switcher等）

### 第二阶段：初步体验（3-5天）
1. 用Trae整理你的桌面文件夹
2. 按照推荐结构创建Obsidian知识库
3. 建立你的第一个知识节点
4. 尝试用Trae生成一篇短文

### 第三阶段：深度探索（1-2周）
1. 用Trae整理素材，导入Obsidian建立关联
2. 查看Obsidian的知识图谱，发现知识联系
3. 用Trae模拟竞争对手，挑战你的想法
4. 开始记录你的思维轨迹

### 第四阶段：持续优化（长期）
1. 每天花10-15分钟更新知识库
2. 每周进行一次红蓝对抗训练
3. 每月回顾知识图谱，分析成长轨迹
4. 定期整理和优化知识库结构

## 🔌 离线支持

### Trae Solo Agent离线支持
| 功能 | 离线支持 | 说明 |
|------|---------|------|
| 本地文件管理 | ✅ 完全支持 | 无需联网即可管理本地文件 |
| 已下载模型运行 | ✅ 支持 | 提前下载模型即可离线使用 |
| 基本文本生成 | ✅ 支持 | 简单文本生成可离线完成 |
| 复杂AI生成 | ❌ 不支持 | 需要联网才能使用高级AI功能 |
| MCP功能 | ⚠️ 部分支持 | 基础MCP功能可离线使用，高级功能需要联网 |

### Obsidian离线支持
| 功能 | 离线支持 | 说明 |
|------|---------|------|
| 核心编辑功能 | ✅ 完全支持 | 所有基础编辑功能均可离线使用 |
| 本地插件 | ✅ 完全支持 | 已安装的本地插件可离线使用 |
| 远程插件 | ❌ 不支持 | 需要联网才能使用远程插件 |
| 知识图谱 | ✅ 完全支持 | 本地知识图谱可离线查看和编辑 |

## 📱 Obsidian推荐插件

### 核心功能插件
1. **Graph View** - 可视化知识图谱，直观展示知识关联
2. **Quick Switcher** - 快速切换笔记，提高操作效率
3. **Backlinks** - 查看笔记的反向链接，发现知识联系
4. **Outline** - 显示笔记大纲，方便导航
5. **Search** - 强大的搜索功能，快速找到所需内容

### 增强功能插件
1. **Templates** - 模板功能，快速创建标准化笔记
2. **Dataview** - 数据视图，实现笔记数据化管理
3. **Canvas** - 画布功能，支持自由布局和思维导图
4. **Excalidraw** - 手绘风格绘图工具，适合创意表达
5. **Templater** - 高级模板功能，支持动态内容生成

### 离线功能支持
- Obsidian核心功能完全支持离线使用
- 大部分本地插件也支持离线运行
- 部分依赖网络的插件（如同步、外部链接预览）需要联网
